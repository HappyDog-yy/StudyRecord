###  2.标签语义

标签的含义，可以让页面结构更加清晰。
#### 2.1 标题标签  

- 分为6个等级`<h1>-<h6>`,head的缩写，代表一到六级标题
- ``` <h1>  我是一级标题 </h1>  ```
#### 2.2 段落标签

- ```<p>这是一段话</p>```
- 在网络标签中，想要分段，必须用到段落标签。空格和`enter`在此处不起作用。
- 一个段落用一对段落标签。  
#### 2.3 换行标签

- 单标签：`<br />`，意为`break`。
- 换行之后，行间距不变；而另起一段，段间距较大。
#### 2.4 文本格式化标签

- 可设置粗体、下划线、斜体
- 加粗：`<strong>`、`<b>`
- 倾斜：`<em>`、`<i>`
- 删除线：`<del>`、`<s>`
- 下划线：`ins`、`u`  
#### 2.5`div`和`span`标签，区块

- 没有语义，一个区块，可以理解为一个盒子，用来装东西 ，用于布局。
- 一个`div`标签独占一行，即一行只能放一个`div`盒子；
- 一行可以放多个`span`盒子。
#### 2.6 图像标签和路径
##### 2.6.1 图像标签及其属性

- 单标签：`<img>`，在所有属性之后加上斜杠
-  不可少的属性：`<src = "url">`：用于指定图像的路径名或者链接 
-  `alt`：图像不能正常显示时替换的文字
-  `title`：提示文本，鼠标放到图像上的文字
-  `width`：图像宽度大小
- `height`：图像高度
- 宽度和高度只修改其中一个，会等比例缩放
- `border`：给图片设定边框
- 属性之间用空格分开
##### 2.6.2 路径

- 相对路径
`./images/img.jpg`、`../04/img.jpg`(上一级目录的04文件夹下的`img.png`)
- 绝对路径
完整路径或者完整的网络地址

### 2.7 链接标签

##### 2.7.1 超链接语法格式：

```
<a>文本或者图像</a>
```

`a`是anchor的缩写，译为锚。

- `herf`：指定链接目标的`url`地址，是超链接不可或缺的属性；
- `target`：用于指定链接页面的打开方式，其中`_self`为默认值，会直接跳转到链接的界面，`_blank`代表在新窗口中打开，原页面保持不变。

##### 2.7.2 超链接分类

- 外部链接，web浏览器的复制过来的，别人的界面
- 内部链接，自己写的`html`文档的路径直接附上
- 空链接`<a herf="#">文本</a>`。
- 下载链接：地址链接的是文件、`.exe`、`zip`
- 所有的网页元素都可以添加超链接，如文本、图像、表格、音频等

下面是为图片添加超链接的示例： 
```
<a herf="链接"><img src="图片地址"></a>
```

##### 2.7.3 锚点链接

可以快速定位到页面中的某个位置，分两步可以实现：
- 在链接文本的`herf`属性中，设置属性值为`#名字`，如`<a herf="#two">第二章</a>`
- 找到目标位置标签，里面添加`id`属性=刚刚的名字，如`<h3 id="two">第二章的内容</h3>`

### 表格

- 整个表格用`table`双标签包裹，包含`thead`和`tbody`两部分
- 表头写在`thaed`中，除表头外的所有行写在`tbody`中
- `tr`代表表格中的一行`table row`，包裹一行的所有列
- 表头的每个列用`th`标签来定义`table header cell`
- 除表头外的每一行用`td`标签来定义`table data cell`

**合并单元格**
- 跨行合并：`rowspan`="个数"，写在最上侧单元格
- 跨列合并：`colspan`="个数"，写在最左侧的单元格
- 注意最后要删除多余的单元格，比如，3列的单元格后两个合并的话，只写两个单元格

### 列表

- 分为有序列表`ordered`、无序列表`unordered`和自定义列表`definte`三种
- 有序列表`ol`包裹，`li`是每个元素的标签
- 无序列表`ul`包裹，`li`是每个元素的标签
- 自定义列表`dl`包裹，`dt`是每个项目的标签，`dd`是每个项目的描述

#### 表单

- 为了收集用户信息
- 表单三要素：表单域、表单控件、提示信息
- 表单域：`<form>`标签定义表单域，
- 表单元素在表单域中写

```
<form action="url地址（接受表单数据的服务器程序的url地址）" method="数据提交方式get/post" name="表单名称">
</form>
```

```
<label for="name">用户名：</label>
<input type="text" id="name" name="name" required>
```

- 其中的for和id对应，代表上面的标签与下面的id一一对应
- required代表提交不能为空

#### 常用type值

- 文本text、密码password、
- 邮箱email、数字number、日期选择器date
- 单选按钮radio、提交按钮submit、文件上传file
